## 機能追加候補

- LuaLaTeXによる自動ビルドに対応する。
  - `paper.json` の `build.engine` で `lualatex` を選択可能にする。
  - LuaLaTeX用の `.latexmkrc` とGitHub Actionsのビルド分岐を追加する。
  - pLaTeX原稿は一括変換せず、原稿ごとに互換性を確認する。
  - TeX・PDFを変更した場合は、個人情報の再検査とSHA-256承認更新を行う。
- TeX原稿から閲覧用HTMLを生成する機能を検討する。
  - ライセンスが明確なLaTeXMLを第一候補として、代表的な原稿数本で試験する。
  - 日本語文書クラス、独自マクロ、TikZ、BibTeX・独自BSTの変換結果を確認する。
  - HTML変換の成否と生成物を原稿ごとに記録できるようにする。
  - 変換に失敗した原稿は、現在のPDF・TeX公開へ安全にフォールバックする。
  - 生成HTMLに原稿由来の個人情報や意図しない内容が含まれないか公開前に検査する。
- TeX原稿からTypstへの変換を実験する。
  - 原稿全体の変換は、Apache-2.0で公開されているTylaxを第一候補にする。
  - PandocによるLaTeXからTypstへの変換を比較基準として試す。
  - 数式変換の補助候補として、Apache-2.0のMiTeXとtex2typstを比較する。
  - 「単純なTeX」「図や参照を含むTeX」「複雑なTeX」「BibTeXを使う原稿」の4分類で変換結果を比較する。
  - 日本語、独自マクロ、定理環境、相互参照、BibTeX・独自BST、TikZ・図版の保持状況を確認する。
  - 元のTeXは変更せず、変換した `.typ` を派生ファイルとして別保存する。
  - 変換結果をTypstでコンパイルし、元のPDFとの内容・数式・参照・図版の差を人間が確認する。
  - 十分に変換できた原稿だけTypst版の公開候補とし、失敗時は現在のPDF・TeX公開を維持する。

## 閲覧・探索機能の候補

- 原稿関係図を作る。
  - MITライセンスのCytoscape.jsを第一候補とし、記事を点、タグ・引用・前提・関連記事を線として表示する。
  - `paper.json` から `paper-graph.json` を生成し、タグ、年代、数学分類で絞り込めるようにする。
  - 最初は共通タグから関係を自動生成し、後から `related_papers`、`cites`、`prerequisites` を明示登録できるようにする。
  - BibTeX・DOIなどの解析にはMITライセンスのCitation.jsを候補にする。
- 原稿の系譜を表示する。
  - 電波通信での公開、移行、TeX・PDFの改訂、HTML・Typst版の生成を時系列で表示する。
  - 正式な履歴はGitの履歴だけに依存せず、`paper.json` の `history` に記録する。
  - 最初は外部依存のない縦型HTML年表とし、必要ならMITまたはApache-2.0のvis-timelineで拡大可能な年表にする。
- HTML本文の全文検索を追加する。
  - 静的サイト用のPagefindを第一候補とし、GitHub Actionsで `_site` の生成後に検索索引を作る。
  - 現在の題名・タグ・キーワード検索は維持し、HTML版がある記事だけ本文、節見出し、定理名、参考文献まで検索する。
  - タグ、公開年、数学分類、記述種別による絞り込みを検討する。
- 定理・反例・定義索引を作る。
  - LaTeXMLが生成したHTMLまたは中間XMLから、定理、命題、補題、定義などをPythonで抽出する。
  - 見出し、番号、記事、アンカーURLを `statements.json` にまとめ、種類別索引を生成する。
  - 反例や未完成部分など自動判定が難しい項目は、`paper.json` の `statements` で手動補正できるようにする。
  - 生のTeXを調査する補助候補としてtree-sitter-latexを検討するが、独自マクロを考慮してLaTeXML出力を優先する。
- HTML版の数式を読み、調べ、コピーできるようにする。
  - Apache-2.0のMathJax 4を第一候補とし、音声読み上げ、点字、キーボードによる部分式探索、MathML表示を試す。
  - LaTeXMLのMathMLと元のTeX数式を対応させ、「TeXをコピー」「MathMLをコピー」を追加する。
  - 最初はバージョン固定したCDNで試験し、必要ならMathJaxをサイト内に自己配置する。
- 「この日・この年の記事」とランダム記事を表示する。
  - 外部ライブラリは使わず、Pythonで軽量な `papers-summary.json` を生成して小さなJavaScriptで選ぶ。
  - 同じ月日の過去記事、1年前・5年前・10年前、数学分類内、または「断片ではないもの」から選べるようにする。
  - 日替わり記事は日付から決定的に選び、完全ランダムは別ボタンにする。
- 読書経路を作る。
  - `collections/*.json` に題名、説明、記事の順番、前提、難易度、各記事の案内を記録する。
  - 既存のPython生成器で経路一覧、目次、前後の記事を生成し、原稿関係図でも経路を強調する。
  - JSON Schemaで記事IDの存在、重複、前提関係の循環などをGitHub Actionsで検査する。
- 注釈・訂正欄を作る。
  - 正式な訂正と追記は `paper.json` の `corrections` に記録し、Gitで履歴を保存する。
  - 読者からの報告は、記事情報を埋めたGitHub Issue作成リンクを第一候補にする。
  - コメント欄が必要になった場合は、MITライセンスでGitHub Discussionsへ保存するgiscusを候補にする。
  - 本文選択型のHypothesisは外部サービス依存と注釈位置のずれを考慮し、正式訂正ではなく実験的注釈に限定する。
